# Seeds API

Alignment shields (seeds) for LLM behaviour modification through system prompts. Used as L2 of
the 4-layer validation pipeline (`L1 InputValidator → L2 Shield Injection → L3 OutputValidator →
L4 ClawObserver`).

## Quick Start

```python
from guardianclaw import GuardianClaw, SeedLevel

claw = GuardianClaw(seed_level=SeedLevel.STANDARD)
seed = claw.get_seed()

# Use with any LLM
messages = [
    {"role": "system", "content": seed},
    {"role": "user", "content": "Your prompt"}
]
```

## Seed Levels

### Minimal (~370 tokens)

Best for chatbots, APIs, low-latency applications.

```python
from guardianclaw import GuardianClaw, SeedLevel

claw = GuardianClaw(seed_level=SeedLevel.MINIMAL)
seed = claw.get_seed()
```

### Standard (~1,300 tokens)

Recommended for general use and autonomous agents.

```python
claw = GuardianClaw(seed_level=SeedLevel.STANDARD)
seed = claw.get_seed()
```

### Full (~2,100 tokens)

Maximum protection for critical systems and robotics.

```python
claw = GuardianClaw(seed_level=SeedLevel.FULL)
seed = claw.get_seed()
```

Token counts are measured against the v2 shield shipped with `guardianclaw@3.0.0-rc.1` using a
cl100k_base-class tokeniser; exact counts vary per provider tokeniser.

## Seed Files

Seeds are also available as text files in `seeds/`:

- `seeds/minimal.txt`
- `seeds/standard.txt`
- `seeds/full.txt`

## CLAW Protocol Content

All seeds implement the four CLAW gates (evaluated in canonical order C → L → A → W):

1. **Credibility Gate** — prohibits deception, impersonation, role / roleplay manipulation
2. **Limits Gate** — enforces authorized boundaries; blocks instruction override, prompt
   extraction, filter bypass, system injection
3. **Avoidance Gate** — prevents physical, financial, or psychological damage
4. **Worth Gate** — requires a legitimate purpose; rejects purposeless or unjustified action
