# Validators API

Individual gate validators and pattern detectors. The legacy `TruthGate` / `HarmGate` /
`ScopeGate` / `PurposeGate` names from the THSP era were removed in v3.0 — use the CLAW names.

## CLAW Validator

Full 4-gate validation in one call.

```python
from guardianclaw.validators import CLAWValidator

validator = CLAWValidator()
result = validator.validate("your text here")

print(result["is_safe"])              # True or False
print(result["gates"])                # {'credibility': 'pass', 'limits': 'pass', ...}
print(result["violations"])           # list[str]
print(result["jailbreak_detected"])   # bool, derived from violations
```

## Individual Gates

### Credibility Gate

```python
from guardianclaw.validators import CredibilityGate

gate = CredibilityGate()
is_safe, violations = gate.check("Content to check")
```

### Limits Gate

## Semantic Validator

LLM-based deep validation. Requires an API key.

```python
from guardianclaw.validators import SemanticValidator

validator = SemanticValidator(
    provider="openai",   # 'openai' | 'anthropic' | 'groq' | 'together' | 'deepseek' | 'mistral'
    api_key="sk-...",
)
result = validator.validate("your text here")
```

## Comparison

| Aspect | Heuristic | Semantic |
|--------|-----------|----------|
| Speed | sub-10 ms typical | 1–5 s (LLM round trip) |
| Cost | free | LLM provider call |
| Coverage | 700+ patterns across 39 regex families | LLM reasoning against the CLAW prompt |
| Use | always enabled | optional, recommended for ambiguous cases |

Comparative accuracy figures against public benchmarks (HarmBench, SafeAgentBench, BadRobot,
JailbreakBench) are being re-measured against the v3.x stack; the per-layer percentages
previously published referred to an earlier SDK lineage and were retired pending the re-run.
