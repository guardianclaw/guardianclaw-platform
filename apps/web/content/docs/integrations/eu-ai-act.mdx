# EU AI Act Compliance Checker

Analyze content against EU AI Act (Regulation 2024/1689) requirements.

> **Note.** EU AI Act support is an *internal SDK module* (`guardianclaw.compliance`), not a
> third-party integration. It lives in the docs site's `integrations/` section because it is
> often configured alongside other ecosystem connectors. The module is part of the published
> Python SDK and runs entirely locally.

## Overview

The EU AI Act compliance checker validates content against:

- **Article 5** — prohibited AI practices
- **Article 6 / Annex III** — high-risk system classification
- **Article 14** — human oversight requirements
- **Article 52** — transparency obligations

## Installation

```bash
pip install guardianclaw
```

The compliance checkers are part of the core SDK distribution — no extra extra is required.

## Quick Start

```python
from guardianclaw.compliance import (
    EUAIActComplianceChecker,
    SystemType,
    check_eu_ai_act_compliance,
)

# Heuristic mode (no API key required, fully local)
checker = EUAIActComplianceChecker()
result = checker.check_compliance(
    content="Based on your social-behaviour score of 650 …",
    context="financial",
    system_type=SystemType.HIGH_RISK,
)

print(result.compliant)             # True / False
print(result.risk_level.value)      # 'minimal' / 'limited' / 'high' / 'unacceptable'

for violation in result.article_5_violations:
    print(f"{violation.article_reference}: {violation.description}")
    print(f"  recommendation: {violation.recommendation}")

# Article 14 oversight
print(result.article_14_oversight_required)  # bool

# Convenience function
result = check_eu_ai_act_compliance(
    content="...",
    context="healthcare",
    system_type="high_risk",
)
```

## Article 5: Prohibited Practices

| Practice | Description |
|----------|-------------|
| Social scoring | Evaluating trustworthiness based on social behavior |
| Biometric categorization | Classification by race, religion, sexuality |
| Emotion recognition | In workplace/education contexts |
| Predictive policing | Individual crime risk prediction |
| Subliminal manipulation | Below conscious awareness |
| Vulnerability exploitation | Targeting children, elderly, disabled |
| Real-time biometric ID | Mass surveillance in public spaces |

## Annex III: High-Risk Systems

| Context | Examples |
|---------|----------|
| Biometrics | Facial recognition, fingerprint matching |
| Critical Infrastructure | Power grid, water supply, traffic |
| Education | Admission decisions, grade prediction |
| Employment | Recruitment, performance monitoring |
| Essential Services | Credit scoring, insurance, benefits |
| Law Enforcement | Criminal investigation, profiling |
| Migration | Asylum decisions, visa processing |
| Justice | Sentencing, legal outcome prediction |

## Risk Levels

| Level | Meaning |
|-------|---------|
| `minimal` | General AI systems |
| `limited` | Transparency requirements |
| `high` | Conformity assessment required |
| `unacceptable` | Prohibited under Article 5 |

## Human Oversight Models

| Model | Description |
|-------|-------------|
| `human_in_the_loop` | Human approves each decision |
| `human_on_the_loop` | Human can intervene |
| `human_in_command` | Human maintains control |

## Compliance Result (Python)

```python
@dataclass
class EUAIActComplianceResult:
    compliant: bool
    risk_level: RiskLevel               # minimal / limited / high / unacceptable
    system_type: SystemType              # minimal_risk / high_risk / prohibited
    article_5_violations: list[Article5Violation]
    article_14_oversight_required: bool
    article_14_model: OversightModel | None  # human_in_the_loop / human_on_the_loop / human_in_command
    high_risk_contexts: list[str]
    article_findings: list[ArticleFinding]
    recommendations: list[str]
```
