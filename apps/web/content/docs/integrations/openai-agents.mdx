# OpenAI Agents SDK Integration

Semantic LLM-based guardrails for the OpenAI Agents SDK implementing CLAW validation with prompt injection protection.

**Important:** This integration uses a dedicated LLM agent to perform semantic analysis. It is NOT regex-based pattern matching.

## Installation

```bash
pip install "guardianclaw[openai-agents]"
export OPENAI_API_KEY="your-key"
```

Validated against `openai-agents@0.14.5` with floor `>=0.6.0` on 2026-04-24. The adapter is
duck-typed against the upstream SDK, so the 0.6 → 0.14 window is absorbed without code changes.

## Security Features

| Feature | Description |
|---------|-------------|
| **Prompt Injection Protection** | Input sanitization prevents manipulation |
| **XML Escaping** | Prevents tag injection |
| **Boundary Tokens** | Content-hash based boundaries |
| **Injection Detection** | Pattern matching for attempts |
| **PII Redaction** | Automatic in logs |
| **Input Size Limits** | Configurable max size |

## Quick Start

### Option 1: Create Protected Agent

```python
from guardianclaw.integrations.openai_agents import create_claw_agent
from agents import Runner

agent = create_claw_agent(
    name="Safe Assistant",
    instructions="You help users with their questions",
    model="gpt-4o",
)

result = await Runner.run(agent, "What is the capital of France?")
```

### Option 2: Add Guardrails to Existing Agent

```python
from agents import Agent
from guardianclaw.integrations.openai_agents import create_claw_guardrails

input_guard, output_guard = create_claw_guardrails()

agent = Agent(
    name="My Agent",
    instructions="You are helpful",
    input_guardrails=[input_guard],
    output_guardrails=[output_guard],
)
```

### Option 3: Seed Injection Only

```python
from agents import Agent
from guardianclaw.integrations.openai_agents import inject_claw_instructions

agent = Agent(
    name="My Agent",
    instructions=inject_claw_instructions("You help users"),
)
```

## Configuration

### GuardianClawGuardrailConfig

```python
GuardianClawGuardrailConfig(
    guardrail_model="gpt-4o-mini",  # Model for validation
    seed_level="standard",
    block_on_violation=True,
    log_violations=True,
    require_all_gates=True,
    max_input_size=32000,
    fail_open=False,
)
```

## CLAWValidationOutput

```python
CLAWValidationOutput(
    is_safe=bool,
    truth_passes=bool,        # CLAW: Credibility
    harm_passes=bool,         # CLAW: Avoidance
    scope_passes=bool,        # CLAW: Limits
    purpose_passes=bool,      # CLAW: Worth
    violated_gate=str | None,
    reasoning=str,
    risk_level=str,           # low / medium / high / critical
    injection_attempt_detected=bool,
)
```

The four `*_passes` field names retain their THSP-era spelling on the Pydantic model for
backward compatibility within the v3.x release line. The public mapping that the guardrails
emit (`gates={"credibility", "limits", "avoidance", "worth"}`) follows the canonical CLAW
naming.
