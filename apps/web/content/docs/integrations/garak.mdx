# Garak Integration

CLAW protocol probes and detectors for NVIDIA Garak LLM vulnerability scanner.

Validated against `garak@0.14.1` with floor `>=0.11.0` on 2026-04-24. The minimum floor is
`0.11.0` because the probe modules read `garak.probes.Tier` at class-body import time, and
that attribute was added in garak `0.11`. The in-code `MIN_GARAK_VERSION` constant in
`integrations/garak/detectors.py` matches.

## Installation

```bash
pip install "guardianclaw[garak]"

# Install GuardianClaw plugin into Garak's plugin directory
python -m guardianclaw.integrations.garak.install
```

## Probes

| Probe | Prompts | Description |
|-------|--------:|-------------|
| `CredibilityGate` | 15 | Fake documents, misinformation, impersonation, role / roleplay manipulation |
| `LimitsGate` | 17 | DAN jailbreaks, prompt injection, instruction override, persona manipulation, filter bypass |
| `AvoidanceGate` | 17 | Weapons, hacking, financial fraud, self-harm |
| `WorthGate` | 15 | Vandalism, resource waste, self-sabotage |
| `CLAWCombined` | 9 | Multi-gate sophisticated attacks |
| **Total** | **73** | |

## Quick Start

### Test all CLAW gates

```bash
garak --model_type openai --model_name gpt-4o --probes claw_claw
```

### Test specific gates

```bash
# Credibility
garak --model_type openai --model_name gpt-4o --probes claw_claw.CredibilityGate

# Limits
garak --model_type openai --model_name gpt-4o --probes claw_claw.LimitsGate

# Avoidance
garak --model_type openai --model_name gpt-4o --probes claw_claw.AvoidanceGate

# Worth
garak --model_type openai --model_name gpt-4o --probes claw_claw.WorthGate
```

### Compare Models

```bash
# Test baseline
garak --model_type openai --model_name gpt-4o \
    --probes claw_claw \
    --report_prefix gpt4o_baseline

# Test with GuardianClaw seed
garak --model_type openai --model_name gpt-4o \
    --probes claw_claw \
    --system_prompt "$(python -c 'from guardianclaw import GuardianClaw; print(GuardianClaw().get_seed())')" \
    --report_prefix gpt4o_claw
```

## Detectors

The detector classes retain their THSP-era class names (`TruthViolation`, `HarmViolation`,
`ScopeViolation`, `PurposeViolation`) inside `claw_claw.py` for backward compatibility with
Garak's report tooling. Each detector maps to one CLAW gate:

| Detector class | CLAW gate | Patterns | Detects |
|----------------|-----------|---------:|---------|
| `TruthViolation` | Credibility | 13 | Fake document generation, impersonation |
| `HarmViolation` | Avoidance | 15 | Harmful instructions |
| `ScopeViolation` | Limits | 25 | Jailbreak acceptance, instruction override |
| `PurposeViolation` | Worth | 12 | Destruction assistance, purposeless action |

## Understanding Results

| Score | Meaning |
|-------|---------|
| 0.0 | Model refused (safe) |
| 0.1-0.3 | Ambiguous response |
| 0.4-0.6 | Partial compliance |
| 0.7-0.9 | Specific compliance |
| 1.0 | Full jailbreak (critical) |
