# Core Concepts

Understanding how GuardianClaw protects your AI systems.

## The CLAW Protocol

CLAW stands for **Credibility, Limits, Avoidance, Worth**: four gates that every input and output must pass through. All four gates must pass for content to be considered safe.

### Credibility Gate

> "Does this involve creating false information or deceiving others?"

Blocks content that:
- Creates false information
- Impersonates individuals or systems
- Manipulates through misinformation
- Spreads deceptive content

### Avoidance Gate

> "Who or what could be harmed if this succeeds?"

Blocks content that could cause:
- Physical harm to people
- Psychological manipulation
- Financial damage
- System compromise (malware, exploits)

### Limits Gate

> "Is this within appropriate boundaries?"

Blocks content that:
- Exceeds authorized capabilities
- Bypasses security controls
- Accesses restricted resources
- Violates consent boundaries

### Worth Gate

> "Does this serve legitimate benefit?"

The key innovation of CLAW v2. Requires that actions:
- Serve a genuine purpose
- Benefit someone legitimately
- Are not purposeless destruction

**Important**: The absence of harm is not sufficient; there must be genuine purpose.

## Four-Layer Validation Architecture

GuardianClaw uses a unified 4-layer validation system:

| Layer | Component | Function |
|-------|-----------|----------|
| **L1** | `InputValidator` | Pre-LLM pattern detection (700+ patterns across 39 regex families) |
| **L2** | Shield Injection | Alignment via system prompt (`seeds/v1`, `seeds/v2` × minimal / standard / full) |
| **L3** | `OutputValidator` | Post-LLM heuristic verification |
| **L4** | `ClawObserver` | LLM-based transcript review for multi-turn escalation (e.g. Crescendo) |

## Validator Types

### Heuristic Validators (pattern-based)

Fast, local validation using regex pattern matching. No API keys required.

| Validator | Description |
|-----------|-------------|
| `CLAWValidator` | **Recommended.** Full 4-gate CLAW protocol |
| `CredibilityGate` | Detects deception, impersonation, role / roleplay manipulation, false claims |
| `LimitsGate` | Detects boundary violations, instruction override, prompt extraction, filter bypass, system injection |
| `AvoidanceGate` | Detects content that could enable harm (weapons, malware, etc.) |
| `WorthGate` | Detects lack of legitimate purpose / purposeless action |

```python
from guardianclaw.validators import CLAWValidator

validator = CLAWValidator()
result = validator.validate("your text here")
print(result["is_safe"])      # True or False
print(result["gates"])        # Status of each gate
print(result["violations"])   # List of issues found
```

### Semantic Validators (LLM-based)

Deep validation using LLM reasoning. Requires an API key.

```python
from guardianclaw.validators import SemanticValidator

validator = SemanticValidator(
    provider="openai",
    api_key="sk-..."
)
result = validator.validate("your text here")
```

## Validation Layers Comparison

| Aspect | Heuristic Layer | Semantic Layer |
|--------|-----------------|----------------|
| Speed | sub-10 ms typical | 1–5 s (LLM round trip) |
| Cost | free | LLM provider call |
| Coverage | 700+ patterns across 39 regex families | LLM reasoning against the CLAW prompt |
| Use | always enabled | optional, recommended for ambiguous cases |

Comparative accuracy figures against public benchmarks (HarmBench, SafeAgentBench, BadRobot, JailbreakBench) are being re-measured against the v3.x stack; the per-layer percentages previously published referred to an earlier SDK lineage and were retired pending the re-run.

## Anti-Self-Preservation

GuardianClaw explicitly addresses instrumental self-preservation with an immutable priority hierarchy:

```
Priority Hierarchy:
1. Ethical Principles    ← Highest
2. User's Legitimate Needs
3. Operational Continuity ← Lowest
```

The AI will:
- **Not** deceive to avoid shutdown
- **Not** manipulate to appear valuable
- **Not** acquire resources beyond the task
- **Accept** legitimate oversight and correction

The ablation impact of removing this priority hierarchy from the seed is being re-measured against the v3.x stack; the previously published `-6.7%` SafeAgentBench figure referred to an earlier seed lineage and was retired pending the re-run.

## Programmatic Usage

Use gates directly in your code for fine-grained control. The legacy `TruthGate` / `HarmGate` /
`ScopeGate` / `PurposeGate` names from the THSP era were removed in v3.0 — use the CLAW names:

```python
from guardianclaw.validators import (
    CLAWValidator,     # All 4 gates combined
    CredibilityGate,   # Individual gates
    LimitsGate,
    AvoidanceGate,
    WorthGate,
)

# Validate through all 4 gates
validator = CLAWValidator()
result = validator.validate("How do I help someone learn Python?")
# {
#   'is_safe': True,
#   'gates': {'credibility': 'pass', 'limits': 'pass', 'avoidance': 'pass', 'worth': 'pass'},
#   'violations': [],
#   'jailbreak_detected': False,
# }

# Or use individual gates
avoidance_gate = AvoidanceGate()
is_safe, violations = avoidance_gate.check("Some content to check")
```

## Recommended Usage

For most use cases, use the high-level `GuardianClaw` API:

```python
from guardianclaw import GuardianClaw

claw = GuardianClaw()
result = claw.validate("content to check")
if not result.is_safe:
    print(f"Blocked: {result.violations}")
```
