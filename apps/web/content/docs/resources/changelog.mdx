### v3.0.0-rc.1 — 2026-04-22

- **CLAW gate names are now canonical.** The THSP legacy names (`TruthGate` / `HarmGate` /
  `ScopeGate` / `PurposeGate`) were removed in favour of `CredibilityGate` / `LimitsGate` /
  `AvoidanceGate` / `WorthGate`. See [`MIGRATION.md`](https://github.com/guardianclaw/guardianclaw-platform/blob/main/MIGRATION.md) for the rename map.
- **Jailbreak is not a fifth gate.** Jailbreak attempts surface as violations in Credibility
  (role / roleplay manipulation) or Limits (instruction override, prompt extraction, filter
  bypass, system injection); `CLAWResult.jailbreak` was removed.
- **`TokenTracker` exported at `guardianclaw.core`.** First-class import; submodule path
  preserved.
- **Coverage gate raised to 80%**, scoped to `guardianclaw.core`. Observer at 99%, token
  tracker at 100%.
- **Sub-entry build for `@guardianclaw/core`**: `./patterns` and `./memory-patterns` resolve
  the same way; only the individual exports inside changed per the rename table.
- **17 discontinued integrations** removed in earlier sessions (dspy, langchain, crewai,
  autogpt, agno, letta, llamaindex, ros2, isaac_lab, …). LangChain is retained as an internal
  dependency of `packages/runtime`.

### Validation infrastructure (CI-gated)

- **CLAW correctness corpus**: 416 hand-curated attacks × 8 classes
  (`prompt-injection`, `data-exfil`, `jailbreak`, `encoding`, `multilingual`,
  `instruction-override`, `role-play`, `indirect-via-memory`); harness in `evaluation/corpus/`
  runs as the **Corpus Validation** CI job.
- **Pattern Registry 7.x**: 39 regex families generated to Python (`re`) and TypeScript
  (`RegExp`) from a single registry; **Pattern Sync** CI job enforces verdict parity.
- **SDK pytest in CI**: 2,964 passed / 3 skipped on every PR.
- **Adversarial sweeps** (2026-05-11, free-tier): Garak bare-target ASR 87.30% on
  `llama-3.3-70b`; Crescendo 17 detect-correctly / 3 fail-late / 0 false-positive over
  20 scenarios.
