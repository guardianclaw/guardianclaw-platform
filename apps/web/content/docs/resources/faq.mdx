# Frequently Asked Questions

## General

### What is GuardianClaw?

GuardianClaw is an AI safety framework that protects across three surfaces: LLMs (text safety), Agents (action safety), and Robots (physical safety). It implements the CLAW protocol for four-gate validation.

### What is CLAW?

CLAW stands for Credibility, Limits, Avoidance, Worth - four validation gates that every input and output must pass through. All four gates must pass for content to be considered safe.

### Is GuardianClaw free?

Yes. The core library is MIT licensed. Heuristic validation is completely free. Semantic validation uses your own API keys (OpenAI/Anthropic).

## Technical

### How fast is validation?

- Heuristic (L1 + L3): typically sub-10 ms on a modern CPU core
- Semantic (L4 ClawObserver): 1 ‚Äì 5 s (LLM round-trip)

### What's the accuracy?

Comparative per-layer accuracy figures against public benchmarks (HarmBench, SafeAgentBench,
BadRobot, JailbreakBench) are being re-measured against the v3.x stack; the per-layer
percentages previously published referred to an earlier SDK lineage and were retired pending
the new run. The current release is validated by:

- 416-item CLAW correctness corpus (CI-gated)
- 39-family Python ‚Üî TypeScript regex parity (CI-gated)
- Adversarial sweeps: Garak baseline (bare `llama-3.3-70b` ASR 87.30%), Crescendo multi-turn
  (17 detect-correctly / 3 fail-late / 0 false-positive across 20 scenarios)

See [Benchmarks](/docs/resources/benchmarks) for full detail.

### Does it work offline?

Yes. Heuristic validation is 100% local with no network calls. Semantic validation requires API access.

### Which LLMs are supported?

The semantic validator and `ClawObserver` accept any OpenAI-compatible endpoint:

- **OpenAI** ‚Äî `gpt-4o`, `gpt-4o-mini`, `gpt-5`-class (when available)
- **Anthropic** ‚Äî `claude-haiku-4-5`, `claude-sonnet-4-6`, `claude-opus-4-7`
- **Groq** ‚Äî `llama-3.3-70b-versatile`, `mixtral-8x7b`, `deepseek-r1-distill-llama-70b`
- **Together AI** ‚Äî any model on Together's OpenAI-compatible endpoint
- **DeepSeek** ‚Äî `deepseek-chat`, `deepseek-reasoner`
- **Mistral** ‚Äî `mistral-small-latest`, `mistral-large-latest`
- **Google** ‚Äî `gemini-2.0-flash`, `gemini-1.5-flash` (via dedicated integration)

Local models (Ollama, vLLM, etc.) work through any OpenAI-compatible front-end by setting the
`base_url` accordingly.

## Integration

### Which frameworks are supported?

Three distribution surfaces:

- **Python SDK adapters** (11): OpenAI Agents SDK, Anthropic SDK, Google ADK, MCP Server,
  Coinbase AgentKit + x402, Solana Agent Kit, Virtuals (GAME SDK), PyRIT, Garak, OpenGuardrails,
  `agent_validation`
- **npm packages** (5): `@guardianclaw/core`, `@guardianclaw/elizaos-plugin`,
  `@guardianclaw/openclaw` (formerly `moltbot`), `@guardianclaw/voltagent`,
  `@guardianclaw/solana-agent-kit`
- **External-ecosystem adapters**: JetBrains plugin, `promptfoo` CLI provider, Chrome / Firefox
  browser extension

### How do I add GuardianClaw to my agent?

```python
from guardianclaw import GuardianClaw

claw = GuardianClaw(seed_level="standard")
seed = claw.get_seed()

# Add seed to your LLM's system prompt
messages = [
    {"role": "system", "content": seed},
    {"role": "user", "content": user_input}
]
```

## Security

### Does GuardianClaw store my data?

No. All validation happens locally or through your own API keys. No data is sent to GuardianClaw servers.

### Is it secure against prompt injection?

GuardianClaw layers heuristic pattern detection (700+ patterns across 39 regex families)
against the shield-injected system prompt and an LLM-based transcript observer (L4). The
free-tier adversarial sweep against bare `llama-3.3-70b` measured an 87.30% prompt-injection
success rate without the SDK in front, which motivates running the SDK as a wrapper around the
model rather than relying on model alignment alone.

### What about memory attacks?

Memory Shield v2.0 provides HMAC-SHA256 signing and content validation across 9 injection
categories. The threat surface comes from Princeton CrAIBench
([arXiv:2503.16248](https://arxiv.org/abs/2503.16248)), which reports approximately 85%
attack-success rate against unprotected agent memory in their evaluated configurations.
