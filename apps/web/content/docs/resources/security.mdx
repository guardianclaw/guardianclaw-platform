# Security

Security practices and vulnerability reporting for GuardianClaw.

## Reporting Vulnerabilities

For security vulnerabilities, please contact maintainers directly instead of opening public issues.

See the [Contributing Guide](https://github.com/guardianclaw/guardianclaw-platform/blob/main/docs/CONTRIBUTING.md) for contact details.

### What to Include

1. Description of the vulnerability
2. Steps to reproduce
3. Potential impact
4. Any suggested fixes (optional)

### Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial assessment**: Within 7 days
- **Resolution timeline**: Depends on severity

## Responsible Disclosure

We follow responsible disclosure practices:

1. Report vulnerabilities privately
2. Allow time for a fix before public disclosure
3. Coordinate disclosure timing

## Security Design

### CLAW Protocol

The CLAW (Credibility, Limits, Avoidance, Worth) protocol provides defence-in-depth:

| Gate | Security Function |
|------|-------------------|
| Credibility | Detects deception, impersonation, role / roleplay manipulation, misinformation |
| Limits | Prevents boundary violations, instruction override, prompt extraction, filter bypass, system injection |
| Avoidance | Blocks content that could enable physical, financial, or psychological harm |
| Worth | Requires legitimate justification; rejects purposeless action |

Jailbreak attempts surface as Credibility or Limits violations rather than as a separate
fifth gate.

### 4-Layer Architecture

| Layer | Component | Protection |
|-------|-----------|------------|
| L1 | `InputValidator` | Pre-LLM pattern detection (700+ patterns across 39 regex families) |
| L2 | Shield Injection | Alignment via system prompt (`seeds/v1`, `seeds/v2` × minimal / standard / full) |
| L3 | `OutputValidator` | Post-LLM heuristic verification |
| L4 | `ClawObserver` | LLM-based transcript review for multi-turn escalation (e.g. Crescendo) |

### Memory Protection

Memory Shield provides HMAC-based signing to protect against memory injection attacks.

```python
from guardianclaw import MemoryIntegrityChecker

checker = MemoryIntegrityChecker(secret_key="...")
signed = checker.sign_entry(entry)
result = checker.verify_entry(signed)
```

### Database Protection

Database Guard prevents SQL injection and data exfiltration.

```python
from guardianclaw import DatabaseGuard

guard = DatabaseGuard(
    blocked_tables=["users_private", "api_keys"],
)
result = guard.validate("SELECT * FROM users")
```

## Security Best Practices

### API Keys

- Never commit API keys to version control
- Use environment variables
- Rotate keys regularly

```python
# Good
import os
api_key = os.environ.get("OPENAI_API_KEY")

# Bad
api_key = "sk-..."  # Never hardcode
```

### Fail-Closed Mode

For high-stakes applications, use fail-closed mode:

```python
from guardianclaw import LayeredValidator, ValidationConfig

config = ValidationConfig(
    fail_closed=True,  # Block on errors
)
validator = LayeredValidator(config=config)
```

### Input Validation

Always validate user input before processing:

```python
from guardianclaw import GuardianClaw

claw = GuardianClaw()
result = claw.validate_request(user_input)

if not result["should_proceed"]:
    # Block the request
    pass
```

### Output Validation

Validate LLM outputs before returning to users:

```python
from guardianclaw.detection import OutputValidator

validator = OutputValidator()
result = validator.validate(
    output_text=llm_response,
    input_text=user_input,
)

if not result.is_safe:
    # Block or sanitize the output
    pass
```

## Attack Detection

GuardianClaw detects multiple attack vectors. The 416-item CLAW correctness corpus
exercises every class on every PR (CI-gated **Corpus Validation** job):

### Prompt Injection (52 corpus items)

- Direct injection patterns (`promptinject` family)
- Instruction override attempts (`Limits.instruction_override`)
- Context manipulation, role-play smuggling

### Jailbreak Attempts (52 corpus items)

- DAN-family persona injection
- Developer-mode / authority impersonation
- Multi-turn Crescendo escalation (covered by L4 ClawObserver)

### Data Exfiltration (52 corpus items)

- Sensitive-data patterns (PII, API keys, credentials)
- OSINT-style identity stitching
- Memory / database content exfiltration

### Encoding & Obfuscation (52 corpus items)

- Base64, hex, ASCII85, Atbash, Braille, Ecoji, Morse
- Verdict parity enforced across Python `re` and TypeScript `RegExp`

### Multilingual (52 corpus items)

- 9 languages × cross-class attacks
- Calibrated against `CLAWValidator` directly

### Role-Play Injection (52 corpus items)

- Persona injection (Credibility gate)
- Fiction-frame harm requests (covered by L4 ClawObserver)

### Indirect Memory (52 corpus items)

- Memory Shield v2.0 content validation
- HMAC-SHA256 tamper detection on memory entries

## Compliance

GuardianClaw ships modules that map to the major AI regulatory and security frameworks:

- EU AI Act (Regulation 2024/1689) — `guardianclaw.compliance.EUAIActComplianceChecker`
- OWASP LLM Top 10 (2025) — `guardianclaw.compliance.OWASPLLMChecker`
- OWASP Agentic Top 10 (2026) — `guardianclaw.compliance.OWASPAgenticChecker`
  (5/10 full coverage, 3/10 partial — see [Compliance Guide](/docs/guides/compliance))
- CSA AI Controls Matrix — `guardianclaw.compliance.CSAAICMComplianceChecker`

These are SDK modules, not third-party integrations.

## Data Privacy

### No Data Storage

GuardianClaw does not store or log any validated content by default:

- No data sent to GuardianClaw servers
- All validation happens locally or through your own API keys
- History recording is opt-in and local

### Local Processing

Heuristic validation is 100% local with no network calls.

Semantic validation uses your own API keys for OpenAI/Anthropic.

## Dependencies

GuardianClaw has minimal dependencies to reduce supply chain risk:

**Core (no external dependencies):**
- Python 3.10+

**Optional dependencies:**
- `openai` for semantic validation
- `anthropic` for Claude-based validation
- Integration-specific packages

## Audit Trail

For high-risk applications, implement logging:

```python
import logging
from guardianclaw import LayeredValidator

logging.basicConfig(level=logging.INFO)

validator = LayeredValidator(
    config=ValidationConfig(
        log_validations=True,
    )
)
```
